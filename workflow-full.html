1

Onboarding

Customer + Admin
Shopify OAuth → Catalog assigned → References uploaded → Output spec locked

Catalog source

  • Admin sets catalog
  • Customer selects their own
  • No catalog? AI infers from store

Locked spec

  • Dimensions, format, color profile
  • Customer done — app takes over
2

Image Ingestion

Shopify
Webhook: new/updated or On-demand request → Pulled from Shopify → Organized + queued

Organization

  • Grouped by SKU
  • Tagged by shot type
  • Duplicates skipped
  • Missing angles flagged

Corrupt images?

  • Quarantined, admin notified
  • Rest continues
3

AI Analysis

AI
Each image measured against catalog + customer references

AI checks

  • Background issues
  • Color accuracy
  • Defects (dust, wrinkles, reflections)
  • Framing & alignment
  • Resolution
  • Lighting & shadows
  • Overall deviation score

Confidence gate

  • High (≥0.80) → auto proceed
  • Medium (0.50–0.79) → coordinator reviews
  • Low (<0.50) → manual triage
4

Package Matching

Rules Engine
Deterministic rules — not AI — map facts to services

Auto — runs immediately

  • Background removal
  • Resolution upscale
  • Straightening & crop
  • Resize to spec
  • Basic color normalization

Human — needs editor

  • Image retouch
  • Ghost mannequin
  • Shadow add / edit
  • Complex color work
Auto lane: no approval needed. Human lane: assigned to editor.
5

Automation Run

Auto
Background cutout → Upscale → Straighten + crop → Resize → Color normalize → Saved as v1
No human services needed? → skip to QC. Otherwise → editor assignment.
6

Editor Assignment

Human

Assignment by

  • Skill match (specialist per job type)
  • Workload capacity
  • Priority

Brief includes

  • Original + auto-processed image
  • AI findings with regions highlighted
  • Remaining services checklist
  • Catalog + references side by side
  • Output spec
7

Editing

Editor
Review auto output → Fix errors if any → Apply human services → Submit final

Editor does

  • Checks automation — corrects if wrong
  • Retouch, ghost mannequin, shadow, color

Versioning

  • Original never overwritten
  • Submitted as versioned edit
8

AI QC

AI
Same model that did analysis checks the edited result

Checks against

  • Original issues — were they fixed?
  • Catalog standard
  • Customer references
  • Batch consistency

Score

  • ≥ 90 → passes → delivery
  • < 90 → fails → rework
Rework: exact issues + region annotations + reference → editor. Cap: 3 cycles. Still failing → human QC reviewer.
9

Delivery

Shopify
Written to Shopify (GraphQL) → Replaces original → Customer notified → Before/after logged

On success

  • In-app + email notification
  • Delivery record saved

Write-back fails

  • Auto-retry 3×
  • Still failing → admin alerted
10

Customer Revision

Customer + Editor
Select image → Annotate region → Pick category → Add comment → Sent to editor

Customer submits

  • Select specific image
  • Mark region on image
  • Category (shadow, color, bg, retouch)
  • Free text comment

Editor + QC

  • Editor gets annotation + context
  • QC: request addressed? New issues? Still meets standard?
  • Passes → delivered. Fails → rework (3 cycles)

Technical Phase Specifications
1

Onboarding (Install + Spec Setup)

Setup
Shopify OAuth PostgreSQL AWS S3

Database Writes

  • Save Merchant metadata & domain
  • Save encrypted Shopify access token
  • S3 file paths & spec records locked to Merchant ID

Server & API Actions

  • Execute OAuth handshake exchange
  • Register Shopify webhook listeners
  • Push catalog reference specs to S3 storage
In Plain Terms

Think of Shopify OAuth like clicking "Login with Google". Instead of giving the app your password, Google confirms who you are and hands over a key (access token) that only unlocks specific parts of your account. Same here, Shopify hands over a digital key to manage the store catalog.

2

Image Ingestion

Shopify
Shopify Webhooks GraphQL API BullMQ Redis

Database Writes

  • Write batch tracking record
  • Write individual image_job records
  • Set status defaults to QUEUED_FOR_ANALYSIS

Server & API Actions

  • Verify security signature on webhook payloads
  • Query Shopify GraphQL product endpoints
  • Enqueue job array to Redis BullMQ worker list
In Plain Terms

A queue is like a deli counter ticket dispenser. Each image gets a ticket number. Workers handle tickets in order. If the system crashes, ticket numbers aren't lost — they sit safely in Redis waiting for workers to restart.

3

AI Analysis & Confidence Gate

AI
OpenAI Vision API PostgreSQL JSON Schema

Database Writes

  • Save raw structured AI findings to job row
  • Update status: ANALYZED, TRIAGE_PENDING, or MANUAL_REVIEW

Server & API Actions

  • Assemble images + references in OpenAI vision call
  • Verify JSON facts against validation schema
  • Apply score logic thresholds (0.80 / 0.50 gates)
View JSON Facts Payload 
{
  "background_is_white": false,
  "color_cast": "warm",
  "defects": ["dust", "wrinkle"],
  "confidence": 0.85
}
In Plain Terms

The worker is like an assistant who picks up a file (image), hands it to an inspector (AI) to check, and files the report (JSON) in the cabinet (database). The AI doesn't decide what happens next; code evaluates the score and routes the job.

4

Package Matching

Rules Engine
Rules Engine PostgreSQL Node.js Logic

Database Writes

  • Save service_selection arrays
  • Update status to READY_FOR_AUTOMATION

Server & API Actions

  • Read analyzed facts from database
  • Execute deterministic rule checks (plain code)
  • Divide selections into autoServices & humanServices
In Plain Terms

This is a plain code checklist. There is no AI involved. If X is true, choose Y. It is predictable and fast, always generating identical outputs for identical input facts.

5

Automation Run

Auto
sharp.js library AWS S3 Cutout / Upscale APIs

Database Writes

  • Save paths of files to S3 against job ID
  • Update status: QUEUED_FOR_EDITOR or QUEUED_FOR_QC

Server & API Actions

  • Call Cutout & Upscale APIs for pixel-level fixes
  • Invoke sharp.js to crop, straighten, and resize
  • Save final automated output to S3 as version_1
In Plain Terms

Sharp.js is a very fast image processing library for Node.js. Think of it like automated, server-run Photoshop that handles resizing and format changes in milliseconds without a human UI.

Editing

Editor
React Dashboard AWS S3 Photoshop Integration

Database Writes

  • Save paths of files to S3 against job ID
  • Update status to QUEUED_FOR_QC

8

AI QC Verification

AI
OpenAI Vision API Rework Logic PostgreSQL

Database Writes

  • Save QC score, failure checklist & coordinates
  • Create rework_request log if checks fail
  • Update status: READY_FOR_DELIVERY or QUEUED_FOR_EDITOR

Server & API Actions

  • Send original, edited, catalog, & specs to vision API
  • Evaluate score values against standard threshold (90)
  • Route based on rework cycle counter limits (3x cap)

9

Delivery

Shopify
Shopify GraphQL BullMQ Retries Sendgrid API WebSockets

Database Writes

  • Save delivery timestamps & Shopify transaction IDs
  • Save final comparative before/after reference links
  • Update status to DELIVERED (or alert flag if failed)

Server & API Actions

  • Call Shopify GraphQL mutation productCreateMedia
  • Overwrite the original target catalog media URL
  • Dispatch socket alerts and email notifications
In Plain Terms

If Shopify's API is temporarily down, the worker fails gracefully. BullMQ automatically retries the delivery up to 3 times with exponential backoff delays. If it still fails, it flags a warning on the admin dashboard while keeping the file safely stored in S3.

Database Status Trail Index

The complete status lifecycle tracked in PostgreSQL for every image job.

1 QUEUED_FOR_ANALYSIS

Job created and queued in BullMQ. Waiting for an AI analysis worker to pick it up.

2 ANALYZING

An AI worker has picked up the job and is calling the OpenAI Vision API with the image and references.

3 ANALYZED

AI analysis is complete. Structured JSON facts describing detected features are saved to the database.

4 PACKAGE_MATCHED

Rules engine has mapped detected facts to services, dividing them into auto vs human task lists.

5 QUEUED_FOR_AUTOMATION

Enqueued to automated image workers for cutout and cropping tasks.

6 AUTOMATION_RUNNING

The sharp.js code engine is executing automated cutouts, resizing, and upscaling tasks.

7 QUEUED_FOR_EDITOR

Requires manual correction (e.g. mannequin/shadows). Assigned to an editor's inbox brief.

8 IN_EDITING

The assigned designer has checked out the brief and is currently editing the image.

9 QUEUED_FOR_QC

Designer upload complete. Image is placed in the QC worker queue for automated checking.

10 QC_RUNNING

AI vision checks the edited version against original defect arrays and style catalogs.

11 READY_FOR_DELIVERY

Image passed quality checks (score ≥ 90) and is queued for Shopify sync workers.

12 DELIVERING

GraphQL media synchronization with the Shopify merchant store catalog is currently running.

13 DELIVERED

Shopify write-back successful. Merchant alerted via Sendgrid and socket notifications.

14 REVISION_REQUESTED

Merchant rejected the delivery. Visual annotations are logged and the job is returned to editing.
