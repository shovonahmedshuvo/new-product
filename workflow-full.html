Technical Phase Specifications

Phase 1 — Onboarding
Stack
Shopify OAuth · PostgreSQL · AWS S3
Database Writes
Customer metadata & domain · encrypted Shopify token · S3 paths locked to Customer ID
Server & API Actions
OAuth handshake exchange · register Shopify webhook listeners · push catalog specs to S3
Shopify OAuth is like “Login with Google” — Shopify hands over a digital key that only unlocks specific parts of the store, no password shared.
Phase 2 — Image Import
Stack
Shopify Webhooks · GraphQL API · BullMQ · Redis
Database Writes
Write batch tracking record · write image_job records · set status defaults to QUEUED_FOR_ANALYSIS
Server & API Actions
Verify security signature on webhook payload · query Shopify GraphQL product endpoints · enqueue job array to Redis BullMQ
A queue is like a deli counter ticket dispenser — each image gets a number; workers handle tickets in order. Numbers survive crashes.
Phase 3 — AI Analysis & Confidence Gate
Stack
OpenAI Vision API · PostgreSQL · JSON Schema
Database Writes
Raw AI findings saved to job row · status: ANALYZED / TRIAGE_PENDING / MANUAL_REVIEW
Server & API Actions
Images & refs to OpenAI Vision · validate JSON schema · apply score thresholds (0.80 / 0.50 gates)
View JSON Facts Payload 
{
  "background_is_white": false,
  "color_cast": "warm",
  "defects": ["dust", "wrinkle"],
  "confidence": 0.85
}
The AI inspects and files a structured report (JSON). Code evaluates the score and routes the job — the AI does not decide what happens next.
Phase 4 — Package Matching
Stack
Rules Engine · PostgreSQL · Node.js Logic
Database Writes
Save service_selection arrays · update status to READY_FOR_AUTOMATION
Server & API Actions
Read analyzed facts · execute deterministic rule checks · divide selections into autoServices & humanServices
Plain code checklist — no AI involved. Identical inputs always produce identical outputs. Predictable and fast.
Phase 5 — Automation Run
Stack
sharp.js · AWS S3 · Cutout & Upscale APIs
Database Writes
Save S3 file paths against job ID · update status to QUEUED_FOR_EDITOR or QUEUED_FOR_QC
Server & API Actions
Cutout & Upscale APIs for pixel-level fixes · sharp.js to crop, straighten & resize · save output to S3 as version_1
sharp.js is automated server-run Photoshop — resizing and format changes in milliseconds, no human UI required.
Phase 6 — Editor Assignment
Stack
Assignment Algorithm · PostgreSQL
Database Writes
Log designer assignment row · save Compiled Editor Brief JSON · update status to QUEUED_FOR_EDITOR
Server & API Actions
Query active workloads & skill categories · match job to best designer · compile images, refs & spec sheets into brief
Phase 7 — Editing
Stack
React Dashboard · AWS S3 · Photoshop Integration
Database Writes
Save S3 file paths against job ID · update status to QUEUED_FOR_QC
Server & API Actions
Read brief & stream image buffers to UI · editor manual corrections (Photoshop) · upload edited output to S3 as version_2
The React dashboard reads and writes database records. Uploading sends the file directly to AWS S3 — the database stores only the resulting link.
Phase 8 — AI QC Verification
Stack
OpenAI Vision API · Rework Logic · PostgreSQL
Database Writes
Save QC score, failure checklist & coordinates · create rework_request log if checks fail · status: READY_FOR_DELIVERY or QUEUED_FOR_EDITOR
Server & API Actions
Send original + edited + catalog + specs to vision API · evaluate against threshold (score 90) · route by rework counter (3x cap)
View JSON QC Result Payload 
{
  "score": 87,
  "pass_fail": "fail",
  "failed_checks": ["shadow_unnatural", "edge_halo"],
  "feedback": [
    "Shadow direction wrong -- should angle right",
    "Halo on upper-left collar edge -- refine cutout"
  ],
  "regions": [{ "label": "collar", "x": 120, "y": 60 }]
}
Phase 9 — Delivery
Stack
Shopify GraphQL · BullMQ Retries · Sendgrid · WebSockets
Database Writes
Save delivery timestamps & Shopify transaction IDs · before/after reference links · update status to DELIVERED
Server & API Actions
Call Shopify GraphQL productCreateMedia mutation · overwrite catalog media URL · dispatch socket alerts & email via Sendgrid
BullMQ retries delivery up to 3x with exponential backoff if Shopify is temporarily down. On final failure a dashboard warning fires; S3 keeps the file safe.
Phase 10 — Customer Revision Request
Stack
Annotation Canvas · API Route · BullMQ Loop
Database Writes
Write revision_request record · log coordinates, categories & text notes · update status to QUEUED_FOR_EDITOR (revision)
Server & API Actions
Receive feedback JSON via API endpoint · inject revision metadata into editor brief · run comparative AI check with revision context

  "category": "shadow",
  "annotation": { "x": 200, "y": 150, "note": "shadow looks wrong here" },
  "comment": "The shadow looks unnatural on the left side"
}

Database Status Trail Index

The complete status lifecycle tracked in PostgreSQL for every image job.

1 QUEUED_FOR_ANALYSIS

Job created and queued in BullMQ. Waiting for an AI analysis worker to pick it up.

2 ANALYZING

An AI worker has picked up the job and is calling the OpenAI Vision API with the image and references.

3 ANALYZED

AI analysis is complete. Structured JSON facts describing detected features are saved to the database.

4 PACKAGE_MATCHED

Rules engine has mapped detected facts to services, dividing them into auto vs human task lists.

5 QUEUED_FOR_AUTOMATION

Enqueued to automated image workers for cutout and cropping tasks.

6 AUTOMATION_RUNNING

The sharp.js code engine is executing automated cutouts, resizing, and upscaling tasks.

7 QUEUED_FOR_EDITOR

Requires manual correction (e.g. mannequin/shadows). Assigned to an editor's inbox brief.

8 IN_EDITING

The assigned designer has checked out the brief and is currently editing the image.

9 QUEUED_FOR_QC

Designer upload complete. Image is placed in the QC worker queue for automated checking.

10 QC_RUNNING

AI vision checks the edited version against original defect arrays and style catalogs.

11 READY_FOR_DELIVERY

Image passed quality checks (score ≥ 90) and is queued for Shopify sync workers.

12 DELIVERING

GraphQL media synchronization with the Shopify customer store catalog is currently running.

13 DELIVERED

Shopify write-back successful. Customer alerted via Sendgrid and socket notifications.

14 REVISION_REQUESTED

Customer rejected the delivery. Visual annotations are logged and the job is returned to editing.

Design principles

P1 AI detects, code decides.

Models output facts. The workflow engine chooses actions.

P2 Same model analyzes and QCs.

The checker already knows what fixed should look like.

P3 Catalog is ground truth.

All decisions compare against the customer's catalog and references.

P4 Async by default.

Heavy work runs in queues. The UI only reads state.

P5 The source platform owns media.

Your DB owns workflow, audit, and version history.

P6 Every transition is logged.

No silent status changes. The audit log tells the story.

Service mapping

Background removal Background is not pure white or does not match the catalog.
Resolution upscale Image is below the target size.
Color adjustment Color cast, saturation issue, or mismatch.
Image retouch Dust, wrinkles, scratches, or reflections.
Shadow add or edit Lighting or shadow is missing or unnatural.
Straighten, center, crop Composition is off or the crop is not correct.
Resize Final output does not match the output spec.
Ghost mannequin Used when apparel needs a product-only mannequin-removed result.
