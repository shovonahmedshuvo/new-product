New Product Workflow

How the system works

A user uploads reference rules and source images. AI finds issues, code maps those issues to services, humans edit, AI checks the result, and approved images are delivered back to the app.

Input

Customer source images plus catalog rules and reference examples.

Logic

AI detects facts. Code decides services, quotes, routing, and pass or fail.

Output

Approved images are delivered, logged, and billed.

Read this first

The system is simple if you read it in order: detect, decide, edit, check, deliver.

1. Import

Source images and rules enter the system.

2. Detect

AI reads the image and returns structured facts only.

3. Decide

Code turns those facts into services, pricing, and routing.

4. Route

Auto-applicable services run immediately. Human services are assigned to an editor.

5. Edit

A human editor completes the work with AI help.

6. Check

The same AI checks the edited file against the catalog.

7. Deliver

Approved images are written back to the app or export folder.

8. Loop if needed

If QC fails, the image goes back to editing. After 3 tries, it escalates.

What AI does

It only describes what it sees.

background not white wrinkles visible color mismatch confidence score

What code does

It decides the service, price, routing, retry rule, and final pass or fail.

package matching quote generation rework guard delivery
Simple rule: AI finds facts, code makes decisions, humans do the editing, AI checks the result.

Where AI is used

AI is used in three main places. The times below are active processing time, not queue wait time.

Stage How AI is used When it runs Approx time
Analysis Reads the image against the catalog and returns structured facts only. After import 10 to 45 seconds per image Mini model is faster for bulk.
Editor assist Optional background removal, fill, upscale, or color help for the editor. During editing 5 to 120 seconds per task Depends on the edit type.
QC Checks the edited image against the original issues and the catalog. After upload 10 to 45 seconds per image Same model family as analysis.
Optional tagging Cheap angle or duplicate tagging before full analysis. Before analysis 1 to 10 seconds per image Only used when batching or organizing.

Core phases

Onboarding

Customer installs the app and completes OAuth. Catalog and references are uploaded.

Catalog intake

Active catalog, style references, and output spec are stored as ground truth.

Import and organize

Images are pulled from Shopify, grouped by SKU, tagged, and deduped.

Analysis

AI reads each image against the catalog and returns JSON facts.

Package matching

Deterministic rules map facts to services and generate a quote.

Editor assignment

Jobs are routed to the right human editor with a clear brief.

Editing

Editors work in Photoshop or similar tools, optionally with AI assist.

QC

The same model re-checks the edited file against the catalog.

Delivery

Approved images are resized, formatted, and stored for download or sync.

Billing

Charges recorded after delivery per completed job.

Design principles

P1AI detects, code decides.

Models output facts. The workflow engine chooses actions.

P2Same model analyzes and QCs.

The checker already knows what fixed should look like.

P3Catalog is ground truth.

All decisions compare against the customer's catalog and references.

P4Async by default.

Heavy work runs in queues. The UI only reads state.

P5The source platform owns media.

Your DB owns workflow, audit, and version history.

P6Every transition is logged.

No silent status changes. The audit log tells the story.

Service mapping

Background removalBackground is not pure white or does not match the catalog.
Resolution upscaleImage is below the target size.
Color adjustmentColor cast, saturation issue, or mismatch.
Image retouchDust, wrinkles, scratches, or reflections.
Shadow add or editLighting or shadow is missing or unnatural.
Straighten, center, cropComposition is off or the crop is not correct.
ResizeFinal output does not match the output spec.
Ghost mannequinUsed when apparel needs a product-only mannequin-removed result.

System layers

Layer 5: User UI - standalone web app with a clean scanning surface.
Layer 4: API - control plane endpoints and event webhooks.
Layer 3: Workers - analysis, editing prep, QC, delivery, billing.
Layer 2: AI gateway - one interface for all model calls.
Layer 1: Storage - PostgreSQL, Redis, and S3-compatible object storage.
